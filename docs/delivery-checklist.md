# Checklist de Entrega

## Entregáveis obrigatórios

- [ ] JSON exportado do workflow n8n.
- [ ] Vídeo do agente funcionando via n8n e WhatsApp.
- [ ] Link do repositório no GitHub.
- [ ] URL pública do dashboard.
- [ ] Vídeo do dashboard funcionando integrado ao agente.
- [ ] Print ou link do histórico de conversa com IA.
- [ ] README completo e atualizado.
- [ ] Deploy público do dashboard.
- [ ] Dados reais gravados pelo agente, sem mock estático no dashboard.

## Desafio 1: agente WhatsApp

- [ ] Webhook configurado para receber eventos da Evolution API.
- [ ] Tratamento inicial de mensagens recebidas.
- [ ] Registro de contatos no banco.
- [ ] Registro de conversas no banco.
- [ ] Registro de mensagens de entrada e saída no banco.
- [ ] Integração com Gemini API.
- [ ] Prompt principal do agente documentado.
- [ ] Memória de conversa.
- [ ] Classificação de intents.
- [ ] Regras de resposta para a Clínica Sorriso Vivo.
- [ ] Envio automático de resposta via WhatsApp.
- [ ] Tratamento de erros.
- [ ] Exportação do workflow n8n em JSON.
- [ ] Evidência em vídeo do fluxo funcionando.

## Desafio 2: dashboard web

- [ ] Aplicação Next.js criada em `apps/dashboard`.
- [ ] Conexão com Supabase.
- [ ] Listagem de conversas reais.
- [ ] Visualização de status das conversas.
- [ ] Métricas agregadas.
- [ ] Filtros de conversas.
- [ ] Gráficos.
- [ ] Atualização em tempo real.
- [ ] Página de detalhes da conversa.
- [ ] Deploy público na Vercel.
- [ ] Evidência em vídeo do dashboard integrado ao agente.

## Diferenciais planejados

- [ ] Memória.
- [ ] Intents.
- [ ] Fila ou buffering de mensagens.
- [ ] Imagens.
- [ ] Áudios.
- [ ] Google Agenda.
- [ ] RAG.
- [ ] Realtime.
- [ ] Filtros.
- [ ] Gráficos.
- [ ] Sentimento.
- [ ] Handoff humano.
- [ ] Autenticação.
- [ ] Tema claro e escuro.
- [ ] Testes.
- [ ] CI/CD.
- [ ] Vibe Coding Journal.

## Critérios de qualidade

- [ ] Nenhum segredo versionado.
- [ ] Documentação em português brasileiro.
- [ ] Projeto organizado para evolução incremental.
- [ ] Fluxos e decisões técnicas registrados.
- [ ] Dados exibidos no dashboard vindos do banco usado pelo agente.
- [ ] Repositório limpo e compreensível para avaliadores técnicos.
